    _---------- SQL (Structured Query Language) is a standardized programming lanuage used for managing relational databases. It is widely used for tasks such as querying data, updating data, creating and modifying database structures (tables, indexes, etc.), and controlling access to the data.

MySQL, Oracle, Microsoft Access, and Sybase are all relational database management systems (RDBMS) that use SQL as their primary language for interacting with the database. Here's a brief overview of each:

1. **SQL**: SQL is a standardized language used for managing relational databases, including tasks such as querying data, updating data, creating and modifying database structures (tables, indexes, etc.), and controlling access to the data. SQL is not a specific database system but rather a language used across various database management systems.

2. **MySQL**: MySQL is an open-source relational database management system known for its reliability, ease of use, and scalability. It is commonly used for web applications and is popular among developers due to its compatibility with various platforms and programming languages.

3. **Oracle Database**: Oracle Database is a commercial relational database management system developed by Oracle Corporation. It is one of the most widely used enterprise-level databases in the world, known for its advanced features such as high availability, scalability, security, and comprehensive data management capabilities.

4. **Microsoft Access**: Microsoft Access is a relational database management system developed by Microsoft. Unlike MySQL and Oracle, which are server-based databases, Microsoft Access is a desktop database management system. It is commonly used for small to medium-sized projects and by individuals or small teams who need to create and manage databases without extensive technical knowledge.

5. **Sybase**: Sybase is a relational database management system developed by Sybase, Inc., which was later acquired by SAP. Sybase offers various database products, including Adaptive Server Enterprise (ASE), IQ, and SQL Anywhere. ASE, in particular, is known for its performance, scalability, and reliability. It is commonly used in industries such as finance, telecommunications, and government for transaction processing and data warehousing.

These databases have their own strengths, features, and use cases, and the choice of which one to use depends on factors such as the specific requirements of the application, scalability needs, budget, and existing infrastructure.

---------------------Here are some key differences between MySQL, Oracle, Microsoft Access, and Sybase:

1. **Licensing and Cost**:
   - MySQL: Open-source and free to use, with paid enterprise editions available for additional features and support.
   - Oracle Database: Commercial software with various licensing options, including free versions for development and educational use, but typically involves significant costs for enterprise deployments.
   - Microsoft Access: Part of the Microsoft Office suite, so it's often included in Office licenses. No separate cost for standalone use.
   - Sybase: Commercial software with licensing costs typically associated with enterprise-grade databases.

2. **Scalability and Performance**:
   - MySQL: Known for its scalability and performance, especially for web applications and smaller to medium-sized databases.
   - Oracle Database: Designed for high scalability and performance, suitable for large-scale enterprise environments and mission-critical applications.
   - Microsoft Access: Limited scalability compared to server-based databases like MySQL and Oracle. Best suited for small to medium-sized projects.
   - Sybase: Offers scalability and performance suitable for enterprise-grade applications, particularly in industries like finance and telecommunications.

3. **Features and Functionality**:
   - MySQL: Offers a rich set of features for managing relational databases, including support for transactions, stored procedures, triggers, and indexing.
   - Oracle Database: Provides comprehensive features for enterprise-grade data management, including advanced security, high availability, partitioning, and data warehousing capabilities.
   - Microsoft Access: Provides a user-friendly interface for designing and managing databases, with features for creating forms, reports, and queries. However, it lacks many of the advanced features found in server-based databases.
   - Sybase: Offers features for high-performance transaction processing, data warehousing, and analytics, with support for features like parallel processing, partitioning, and advanced indexing.

4. **Target Audience**:
   - MySQL: Widely used by developers and small to medium-sized businesses for web applications and other projects.
   - Oracle Database: Primarily targeted at large enterprises with complex data management needs and high-performance requirements.
   - Microsoft Access: Mainly used by individuals, small teams, and small businesses that require a simple desktop database solution.
   - Sybase: Commonly used in industries such as finance, telecommunications, and government for high-performance transaction processing and data warehousing.

These are general differences, and the suitability of each database depends on specific project requirements, budget constraints, and existing infrastructure.

-- Create a new database called 'example_db'
CREATE DATABASE IF NOT EXISTS example_db;

-- Switch to the 'example_db' database
USE example_db;

-- Create a new table called 'users'
CREATE TABLE IF NOT EXISTS users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50),
    email VARCHAR(100)
);

-- Insert some data into the 'users' table
INSERT INTO users (name, email) VALUES 
    ('John Doe', 'john@example.com'),
    ('Jane Smith', 'jane@example.com'),
    ('Alice Johnson', 'alice@example.com');

-- Query the 'users' table to retrieve all records
